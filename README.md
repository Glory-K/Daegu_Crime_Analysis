# 대구광역시 자치구 단위 CCTV·인구·용도지역과 범죄 발생 특성 분석

공공데이터를 활용하여 **대구광역시 자치구 단위의 범죄 발생 특성**을 비교하고,  
**CCTV 설치 수준, 인구 구조, 용도지역 특성**과의 관계를 탐색적으로 분석한 프로젝트입니다.

---

## 1. 프로젝트 개요

본 프로젝트는 대구광역시 자치구 단위 공공데이터를 활용하여 지역별 범죄 발생 수준을 비교하고,  
CCTV 설치 수준 및 도시 공간 특성과의 관계를 탐색적으로 분석하는 것을 목표로 합니다.

단순 범죄 건수 비교는 인구 규모가 큰 지역에 불리하게 작용할 수 있기 때문에,  
본 분석에서는 **인구 10만 명당 범죄율**을 중심 지표로 설정하였습니다.

또한 전체 범죄뿐 아니라 **5대 범죄**를 별도로 구분하고,  
CCTV 총량, 면적 대비 CCTV 밀도, 방범 CCTV 비중, 상업지역 비율 등을 함께 고려하여  
지역별 범죄 특성을 다각적으로 비교하였습니다.

---

## 2. 분석 목적

- 대구광역시 자치구별 범죄 발생 수준 비교
- 인구를 고려한 범죄율 기준 지역별 위험 수준 비교
- CCTV 설치 수준과 범죄 특성 간 관계 탐색
- 용도지역 특성과 범죄 발생 패턴 간 관계 탐색
- 공공데이터 전처리 및 통합 분석 파이프라인 구축

- **공간 단위**: 대구광역시 자치구
- **주요 비교 시점**: 2024년
- **보조 분석 시점**: 일부 연도 비교 포함
- **분석 성격**: 탐색적 데이터 분석(EDA) 및 비교 분석

> 본 프로젝트는 상관적·탐색적 분석을 목적으로 하며,  
> 특정 변수 간 인과관계를 직접 증명하는 분석은 아닙니다.

## 6. 데이터 전처리

여러 공공데이터를 함께 활용하기 위해 데이터 구조와 기준을 통일하는 전처리 과정을 수행하였습니다.

### 주요 전처리 내용

- 인구 데이터 문자열 수치형 변환 및 연평균 인구 생성
- 범죄 데이터 다중 헤더 재구성 및 자치구명 표준화
- 5대 범죄 기준 재분류
- 자치구별 CCTV 파일 형식 및 칼럼명 통일
- CCTV 설치 목적 표준화
- 주소 기반 좌표 보완 및 geometry 검증
- 용도지역 데이터 구조 정리
- 경계 데이터 좌표계 통일
- 분석용 파생변수 생성

### 전처리 시 중점적으로 고려한 사항

- 데이터 출처별 구조 차이
- 자치구명 표기 불일치
- 일부 값의 결측 및 보정 필요성
- 좌표 정보의 정합성
- 분석 변수와 검증용 보조 변수 구분

- 자치구별 CCTV 설치 수준 비교

## 8. 주요 결과 요약

본 프로젝트를 통해 다음과 같은 내용을 확인하였습니다.

- 자치구별 범죄 발생 수준은 단순 건수와 인구 보정 범죄율 기준에서 다르게 나타날 수 있음을 확인
- CCTV 총량만으로 지역의 관리 수준을 단정하기 어렵고, 밀도와 설치 목적을 함께 볼 필요가 있음을 확인
- 상업지역 비율이 높은 지역은 범죄 발생 특성이 다르게 나타날 가능성이 있음을 확인
- 공공데이터 분석에서는 결과 해석 이전에 데이터 구조와 품질 검토가 매우 중요함을 확인

> 상세 분석 결과와 시각화는 보고서 및 발표자료에서 확인할 수 있습니다.

---

## 9. 프로젝트를 통해 배운 점

이 프로젝트를 통해 단순히 분석 결과를 도출하는 것보다,  
**여러 출처의 데이터를 공통 기준으로 정리하고 신뢰할 수 있는 형태로 구조화하는 과정**이  
분석의 출발점이라는 점을 배웠습니다.

특히 공공데이터는 출처별 구조와 품질 차이가 크기 때문에,  
전처리와 검증 기준을 명확히 세우는 것이 결과의 신뢰도를 좌우한다는 점을 체감했습니다.

또한 분석 결과를 시각화하고 문서화하는 과정까지 포함해  
데이터 분석은 단순 계산이 아니라 **문제 정의, 데이터 정리, 해석, 전달**이 함께 이루어져야 한다는 점을 경험했습니다.
